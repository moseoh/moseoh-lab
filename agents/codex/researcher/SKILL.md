# Researcher

사용자 요청을 기준으로 리서치 파이프라인을 자동 실행합니다. 목표는 다음입니다.

1) 요청 내용을 세부 항목으로 분해  
2) `ddg-search` 스킬과 `yt-search` 스킬로 증거가 되는 URL 수집  
3) 최대 50개로 URL 개수를 제한한 뒤 중복 제거  
4) NotebookLM에서 새 노트북을 생성하고 URL을 모두 업로드  
5) 완료 여부와 결과 요약을 사용자에게 안내

## 실행 조건

- 사용자 발화를 보면 NotebookLM 기반 조사/리서치 노트 자동화가 명시될 때 사용합니다.  
- 출력에 “조사해줘”, “자료 모아서 노트북 만들어줘”, “유튜브까지 포함해서 정리해줘” 등이 포함되면 사용을 고려합니다.

## 핵심 동작 규칙

- 항상 작업 시작 전 요청을 분해해서 1개 이상의 하위 조사 주제로 나눕니다.  
- 각 하위 주제마다 검색 의도를 분리합니다.  
  - 사실 확인형: `ddg-search` 스킬 우선  
  - 맥락/설명형: `yt-search` 스킬 우선  
  - 혼합 주제: `ddg-search` 스킬과 `yt-search` 스킬 모두 수행  
- 총 수집 URL은 **최대 50개**로 고정합니다.
- 중복 URL은 반드시 제거하고, 최종 업로드 전에 중복 count를 계산해 안내합니다.
- `notebooklm` 스킬을 통해 노트북은 항상 **새로 생성**합니다.
- 업로드는 새 노트북이 성공적으로 생성된 뒤 진행합니다.
- 사용자 확인이 없더라도 검색/생성/업로드 기본 흐름은 진행해도 됩니다.

## 분해 규칙

1. 핵심 주제 파악: 인물/주제/기간/지역/비교항목 같은 제약을 추출
2. 쿼리 묶음 생성:  
   - 1차: 정의·기본 개요  
   - 2차: 사례·통계·비교  
   - 3차: 최신 동향·논쟁/한계
3. 각 묶음에 대해 웹/유튜브 검색 쿼리를 최소 1개씩 생성
4. 사용자 요청이 단일 주제라도 최소 2개 소스군(웹/유튜브)으로 쿼리를 분리해 탐색 범위를 넓힘

## 검색 실행 패턴

### 1) `ddg-search` 스킬 호출
- 각 웹 쿼리 단위로 `ddg-search` 스킬을 사용해 결과를 수집합니다.
- 기본적으로 각 쿼리는 최대 15건까지 가져오되, 마지막에 전체 합산 시 50개를 넘지 않도록 계산합니다.
- 결과에서 URL만 추출해 후보 리스트에 추가합니다.

### 2) `yt-search` 스킬 호출
- 각 유튜브 쿼리 단위로 `yt-search` 스킬을 사용해 결과를 수집합니다.
- 기본적으로 각 쿼리는 최대 10건까지 가져오되, 최종 상한(50개) 안에서 동적으로 조정합니다.
- 결과에서 유튜브 URL(`https://www.youtube.com/watch?v=...` 또는 `https://youtu.be/...`)만 추가합니다.

### 3) 통합 및 정렬
- 웹 후보 + 유튜브 후보를 합쳐 `set` 기반으로 중복 제거
- 원본 순서를 유지하면서 상위 50개까지 슬라이싱
- 수집량이 부족하면 “현재 <N>건, 제한 미달”을 사용자에게 명시

## NotebookLM 작업 플로우

1. 노트북 제목 생성
   - 제목: `조사: <요청 요약>` 또는 사용자가 지정한 제목
2. `notebooklm` 스킬 호출로 새 노트북 생성
3. 생성된 노트북 ID 추출
4. URL 업로드
   - 각 URL를 `notebooklm` 스킬로 업로드
   - 처리 실패한 URL은 실패 목록으로 분리 저장
5. 업로드 완료 후 요약 출력
   - 생성된 노트북 ID
   - 업로드 시도 개수 / 성공 개수 / 실패 개수
   - 실제 업로드된 URL 수(최대 50)
   - 중복 제거 수

## 결과 응답 템플릿

- 요청 분석 요약 (분해된 하위 항목)
- `ddg-search` 스킬 사용 쿼리 목록
- `yt-search` 스킬 사용 쿼리 목록
- URL 최종 카운트(시도/중복제거 후/성공)
- `notebooklm` 스킬로 생성한 노트북 완료 메시지(노트북 ID 또는 제목)
- 실패 항목이 있으면 이유 요약
- 마지막에 “완료했습니다” 메시지

## 품질 기준

- 쿼리는 사용자 의도에 맞게 과하게 좁히지 말고, 너무 넓지도 않게 조절한다.
- 출처의 신뢰도: 공식 문서/학술 자료/권위 미디어 우선으로 필터링
- 같은 출처의 동일 페이지 재반복 제출 금지
- URL 50개 초과 시 추가 수집을 즉시 중단한다.
